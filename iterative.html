Iterative Design Project
What Is Iterative Design?
Iterative design is a design process where products undergo multiple cycles of prototyping, testing, and redesign based on the results of that testing.
Project Goal
The goal of this group project was to develop a prototype based on a description of our chosen startup without looking at their actual website. In the process of designing an interface from scratch, our team learned a great deal about collaborative design, user testing and giving constructive feedback.
Startup Chosen
Welcome to Mesh, a social networking site for remote companies! It allows employees to share the projects they are working on, explore other people's projects, and give each other feedback.

Lo-Fi Wireframes
We took ideas from each of our sketches, and combined them into a single set of wireframes. We felt that the card-based dashboard clearly showed the user's current projects, while the tree structure was a good representation of all projects in the system. We also thought it was important for a project to have its own page and for there to be a page where users could submit feedback for another project.
Hi-Fi Prototypes
Hi-Fi Before Critique
We made an interactive high-fidelity prototype based off the wireframe and presented this mockup for a round of critiques.
Critique and Adjustments
We received some valuable critique feedback and we revised our prototype accordingly.
  • Text hierarchy (changed boldness, used one typeface)
  • Dashboard icons (made notifications vs members clear)
  • Project relationship (added parent project selection for the tree logic
  • Unclear login page (added section describing Mesh)
  • Unclear tree purpose (added text to indicate circles represent project hierarchy)
  • Some lack of clarity in navigation (made buttons match the pages they go to, added breadcrumbs)
Hi-Fi After Critique
User Testing
Using our interactive prototype, we conducted a usability test through a remote user testing service. Since the primary functionality of our app is to share project updates and to give feedback, we instructed the users to complete the following tasks and gathered user testing videos from the service.
Tasks
  1. Create a new project
    1. View your projects
    2. Create a new project
    3. Submit new project
  2. Open up one of your projects to feedback
    1. View your projects
    2. Select a project
    3. Open the project up to feedback
  3. Provide Feedback to someones else's project
    1. View all projects
    2. Find and view someone else's project
    3. Provide Feedback
    4. Submit the feedback form
Post-Task Questions
  1. Which task did you find the hardest? Why?
  2. Did the website's navigation match your Expectations?
  3. What would you change about this website if you could?
  4. How effectives was this website for the given tasks?
Results
Going into user testing, we expected it to be pretty straightforward for users to create a new project and open a project to feedback. We were curious to find out whether users could find other people's projects on our project tree and expected there to possibly be some confusion around the explore page.
Observations
  • The first user made no errors in tasks, did tasks pretty quickly but had trouble differentiating between own and others' projects
  • The second user tried to click the member circles from "My Project" page in order to go to other projects
  • All users had trouble finding the "Explore" page, they did not know that that would lead to seeing all possible projects
  • Users did not know what were "their projects" or "other people's projects". They did not instinctively click on the "i" in order to view the key
Potential Changes
Based on our feedback from user testing, we came up with potential changes to improve our design:
  • Change header label from "explore" to "all projects"
  • Make key for projects tree onscreen or make the info button open by default to make sure people know how the project tree works
Conclusion
All in all, this was a fascinating, informative, and enjoyable project. Ideating, designing, and prototyping a web-based application was a challenging, yet incredibly rewarding process; our team learned a great deal about collaborative design, iterative prototyping, and user testing, and developed skills which will accompany us in all future projects.
