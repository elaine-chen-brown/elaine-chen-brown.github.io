React Development Project
What Is React?
React is an open-source JavaScript library used to dynamically render webpages. It operates on a component model, where components store data in their state and pass data to one another through function calls and upon instantiation.
Project Goal
The goal of this project was to create a list interface using React that supports filtering, sorting, and aggregation. I chose to make a virtual bookshelf application, where a user can filter books by genre and author, sort by publication date, and see the total word count of the books they have selected.
Usability Principles
When designing my site, I considered Jakob Nielsen's 10 usability heuristics:
  • Visibility of System Status
    • Users can see which books have been added to the total word count
    • Users are shown an alert window when they add or remove books from their total word count list

  • Match Between System and Real World
    • Filters mimic a user's conceptual model of a bookstore or library: one first goes to a specific genre or author and then views books sorted alphabetically by title

  • User Control and Freedom
    • Users can both add and remove books from the total word count section

  • Consistency and Standards
    • The total word count section is referred to as "Books List" every time it is referenced.
    • Filter labels match the labels shown on the books in the grid

  • Help Users Recognize, Diagnose, and Recover from Errors
    • Users can undo filters by clicking "All"
    • Users can undo adding a book to their books list by clicking "Remove from Books List"

  • Error Prevention
    • Users are prevented from selecting multiple authors or genres at once, since a book cannot have more than one author and main genre
    • Users are prevented from adding a book to their total word count list more than once and are shown an error message if they try to do so

  • Recognition Rather than Recall
    • The filter bar is clearly visible to the user
    • The "Add to Books List" and "Remove from Books List" buttons are clearly displayed

  • Flexibility and Efficiency of Use
    • A user can either filter for exactly which genre and/or author's books they want to see or explore all the books available by scrolling through the grid

  • Aesthetic and Minimalist Design
    • Only neccessary text is shown, and every piece of text has a purpose

  • Help and Documentation
    • While no instructions section is provided, button and filter labels are imperative calls to action that tell the user their task
